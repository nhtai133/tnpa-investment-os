# Quality Attributes

## Auditability

The system must explain who did what, when, and why for material investment workflows.

## Reliability

Investment records should remain available and consistent. Background ingestion failures should be visible and recoverable.

## Traceability

Users should be able to trace a decision back to sources, notes, thesis versions, approvals, and portfolio context.

## Security

Sensitive investment data requires authenticated access, role-based authorization, server-side permission checks, and secret isolation.

## Maintainability

Domain logic should be testable outside presentation code. Framework decisions should not obscure investment workflow rules.

## Data Integrity

Schemas and validation should protect against orphan records, ambiguous dates, mixed currencies, and silent overwrites of approved material.
